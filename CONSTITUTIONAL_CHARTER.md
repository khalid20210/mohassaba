## 💰 المادة الرابعة: مصفوفة الدمج والرسوم {#article-4}

### **الرسوم المطبقة**

| الحالة | الرسم | الشروط |
|--------|-------|--------|
| دمج من نفس القطاع | 0 ريال | مثلاً: صيدلية + عيادة = 0 ريال |
| توسع تكميلي (نفس القطاع، آلية مختلفة) | 80 ريال | مثلاً: مطعم + مقهى |
| فصل جذري (قطاع مختلف) | 100 ريال | يُجبر على حساب منشأة جديد |
| أنشطة مظلة (سوبر ماركت) | 0 ريال | دمج مجاني |

### **قاعدة السوبر ماركت (المظلة)**

```
السوبر ماركت (>130 متر مربع) يسمح له بدمج:
✓ أقسام الأغذية
✓ أقسام الملابس
✓ أقسام المنظفات
✓ أقسام القرطاسية

كل هذا تحت رقم ضريبي واحد بدون رسوم إضافية!
```

### **مثال على حساب الرسوم**

```
مثال 1: صيدلية تريد إضافة عيادة طبية
من: صيدلية (قطاع صحي)
إلى: عيادة (قطاع صحي)
النتيجة: 0 ريال (نفس القطاع)

مثال 2: مطعم يريد إضافة مقهى
من: مطعم (آلية خدمة طعام)
إلى: مقهى (آلية خدمة مشروبات)
النتيجة: 80 ريال (توسع تكميلي)

مثال 3: متجر أغذية يريد إضافة محل ملابس
من: متجر أغذية (قطاع تجزئة)
إلى: متجر ملابس (قطاع تجزئة)
النتيجة: 0 ريال (نفس القطاع)

مثال 4: متجر يريد مقاولات
من: متجر (قطاع تجزئة)
إلى: مقاولات (قطاع إنشاءات)
النتيجة: 100 ريال (فصل جذري) - حساب منشأة جديد
```

## 🔒 المادة السادسة: الأمان والاستمرارية المطلقة {#article-6}

### **أ. سجل الرقابة المجهري (Enhanced Audit Logs)**

كل نقرة يتم تسجيلها مع:
```
✓ معرف المستخدم
✓ عنوان IP
✓ معرف الجهاز الفريد
✓ نوع المتصفح ونظام التشغيل
✓ وقت الإجراء (بدقة الميلي ثانية)
✓ المدة الزمنية للعملية
✓ القيم القديمة والجديدة
✓ حالة النجاح/الفشل
```

الجداول:
- `enhanced_audit_logs` - السجل الشامل
- `user_sessions_tracking` - تتبع الجلسات
- `security_alerts` - التنبيهات
- `permission_changes` - تغييرات الصلاحيات

### **ب. الاستمرارية المطلقة (Automatic Failover)**

```
عند تعطل أي خادم:
1. Load Balancer يكتشف العطل (في <1 ثانية)
2. الطلبات تُحوّل تلقائياً للخادم الاحتياطي
3. المستخدمون لا يشعرون بأي انقطاع
4. البيانات آمنة (مزامنة فوري مع Replicas)

SLA: 99.99% توفر
= <52 دقيقة تعطل في السنة
```

### **ج. النسخ الاحتياطية والاستعادة**

```
النسخ الاحتياطية التلقائية:
- كل ساعة: نسخة حديثة
- كل يوم: نسخة يومية
- كل أسبوع: نسخة أسبوعية
- كل شهر: نسخة شهرية

وقت الاستعادة: <5 دقائق

الجدول: backups
```

### **د. كسر الحلقة (Circuit Breaker)**

```
عند فشل خدمة ما بشكل متكرر:
1. محاولات أولية ← فشل (5 محاولات متتالية)
2. Circuit Breaker ينتقل لـ OPEN
   (يرفض الطلبات للخدمة المعطلة)
3. انتظار لمدة معينة (timeout)
4. محاولة واحدة في حالة HALF_OPEN
5. إذا نجحت: العودة للـ CLOSED
```

### **ه. تدهور ظريف (Graceful Degradation)**

```
عند فشل ميزة معينة:
بدلاً من: ❌ صفحة خطأ
نقدّم:  ✓ ميزة مبسطة

مثلاً:
- Analytics معطل؟ → عرض البيانات الأساسية
- AI Suggestions معطل؟ → بحث يدوي
- Real-time sync معطل؟ → مزامنة عند الاتصال
```
