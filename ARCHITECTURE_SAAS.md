# نظام المؤسسات SaaS — المعمارية الشاملة

## 📋 نظرة عامة

نظام محاسبة شامل يدعم **500+ منشأة** مستقلة تماماً مع:
- **قاعدة بيانات مركزية** (PostgreSQL) لإدارة المنشآت والمزامنة
- **قاعدة بيانات محلية** (SQLite) على كل جهاز للعمل بوضع أوفلاين
- **196 نشاط** بمنتجاتهم وخدماتهم وحساباتهم المحاسبية
- **مزامنة ثنائية الاتجاه** بدون انقطاع في الخدمة

## 🔄 دورة المزامنة

### السيناريو 1: جهاز متصل بالإنترنت

```
Device (Local SQLite)
        │
        ├─────→ 📡 CHECK CONNECTION
        │       └─→ Central Server (Online ✓)
        │
        ├─────→ 📥 PULL
        │       ├─ آخر تحديث محلي: 2026-05-05 14:30
        │       ├─ طلب من الخادم: "أرسل كل التحديثات منذ الساعة 14:30"
        │       └─ الخادم يرسل:
        │           • 50 سجل منتج جديد
        │           • 10 تحديثات للأسعار
        │           • 5 عمليات حذف
        │
        ├─────→ 🎯 MERGE & RECONCILE
        │       ├─ دمج البيانات محلياً
        │       ├─ فحص النزاعات
        │       └─ حل النزاعات (آخر تعديل يفوز)
        │
        ├─────→ 📤 PUSH
        │       ├─ تغييرات محلية منذ آخر مزامنة:
        │       │  • 30 فاتورة جديدة
        │       │  • تعديلات المخزون
        │       │  • إضافة 5 عملاء جدد
        │       └─ ارفع كل هذا للخادم
        │
        └─────→ ✅ SYNC COMPLETE
```

```
Device (Local SQLite) — بلا اتصال ❌
        │
        ├─────→ 📡 CHECK CONNECTION

        │
        ├─────→ 💾 WORK LOCALLY
        │       ├─ أصدر فواتير محلياً ✓
        │       ├─ أضف منتجات ✓
        │       ├─ تحديث المخزون ✓
        │       ├─ كل العمليات تُحفظ محلياً
        │       └─ ضع كل التغييرات في offline_queue
        │
        └─────→ ⏳ WAIT FOR CONNECTION
                │
                ├─→ اتصال استعاد! 🟢
                │
                └─→ بدء المزامنة التلقائية
                   (الخطوات السابقة 2-4)
```

## 📦 ملفات التهيئة الرئيسية

### 1. التهيئة المركزية (شغّل مرة واحدة فقط)

```bash
python setup_centralized_db.py
```

**الملف:** `setup_centralized_db.py`
- ينشئ: `database/central_saas.db`
- الجداول: system_config, tenants, tenant_databases, activities_definitions, sync_queue, audit_log
- الأنشطة: يحمّل 196 نشاط من `modules/terminology.py`

### 2. التهيئة المحلية (شغّل على كل جهاز)

```bash
python setup_tenant_local.py biz-001 pos-cashier-001
python setup_tenant_local.py biz-001 agent-mobile-001
python setup_tenant_local.py biz-002 pos-cashier-001
```

**الملف:** `setup_tenant_local.py`
- الوسائط:
  - `tenant_key`: معرّف المنشأة (biz-001)
  - `device_id`: معرّف الجهاز (pos-001, agent-001)
- ينشئ: `database/local_{tenant}_{device}.db`
- الجداول:
  - local_sync_state — حالة المزامنة
  - offline_queue — التغييرات المحلية
  - activities_local — 196 نشاط محلي
  - products_local, services_local — المنتجات والخدمات
  - invoices_local — الفواتير المحلية
  - inventory_local — المخزون المحلي

### 3. ملف الإعدادات لكل جهاز

بعد تشغيل `setup_tenant_local.py`، يُنشأ ملف إعدادات:

```json
config/device_biz001_pos001.json
{
  "device_info": {
```

## 🔐 الأمان

### تشفير البيانات

- **في الخادم:** كل منشأة معزولة تماماً
- **بين الأجهزة والخادم:** HTTPS (SSL/TLS)
- **في الأجهزة المحلية:** SQLite Cipher (اختياري)

### المصادقة

- كل جهاز يملك `device_id` فريد
- كل منشأة تملك `tenant_key` فريد
- الخادم يتحقق من كل طلب: `tenant_key + device_id`

### التدقيق

- كل عملية تُسجّل في `audit_log`
- تسجيل: من، ماذا، متى، من أين
- إمكانية الاسترجاع الكامل للبيانات التاريخية

## 🛠️ استكشاف الأخطاء

### المشكلة: جهاز POS بلا اتصال

**الحل:**
```
✓ الجهاز يعمل بوضع أوفلاين تلقائياً
✓ جميع الفواتير تُحفظ محلياً
✓ عند استعادة الاتصال، ستبدأ المزامنة تلقائياً
✓ لا توجد فواتير مفقودة
```

### المشكلة: نزاع في المزامنة (تعديل الأغلب من جهازين)

**الحل:**
```
✓ النظام يكتشف النزاع تلقائياً
✓ يستخدم: آخر تعديل يفوز (last-write-wins)
✓ يسجّل النزاع في sync_conflicts
✓ يمكن للمالك مراجعة النزاعات لاحقاً
```

### المشكلة: فشل المزامنة

**الحل:**
```
✓ أعد المحاولة تلقائياً (حتى 3 مرات)
✓ احفظ الخطأ في offline_queue.sync_error
✓ حاول عند اتصال جديد
✓ لوحة تحكم توضح حالة المزامنة
```

## 📈 التوسع المستقبلي

### من 500 منشأة إلى 5,000

1. **الخادم:**
   - ترقية PostgreSQL إلى Multi-Master Replication
   - إضافة Redis Cache لتسريع المزامنة
   - Load Balancer أمام الـ API

2. **الأجهزة:**
   - تقليل interval المزامنة من 5 دقائق إلى دقيقة
   - إضافة Compression للبيانات المُنقلة

3. **المراقبة:**
   - Monitoring Dashboard (Grafana/Prometheus)
   - Alerts تلقائية عند فشل المزامنة
   - Analytics على الأداء
